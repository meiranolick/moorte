Using MooRTE:

Basic usage of MooRTE just requires calling it: new MooRTE();
This will extend all of the textareas on the page and convert them to rich textareas. 

That's it!!
Here's the whole page.

<!doctype>
<HTML>
    <head>
        <title>MooRTE</title>
        <script src="mootools.js"></script>
        <script src="MooRTE.js"></script>
    </head>
    <body>
        <textarea> some text to edit </textarea>
       
        <script> new MooRTE() </script>
    </body>
</HTML>

The textarea will have a toolbar above it with all of the classic editing keys.

To convert other elements (besides textareas), add an "RTE" class to the element.  It makes no difference whether the element you add "RTE" to was originally editable (such as a textarea) or not (such as a div).  In the example below, the div will have a toolbar above it.

<body>
    <div class="RTE"> some text to edit </div>
    <script> new MooRTE() </script>
</body>

Alternatively, you can pass the elements you wish to enrich into the function using CSS selectors. 
<body>
    <div id="someText"> some text to edit </div>
    <script> new MooRTE('div#someText') </script>
</body>

Mootools supports virtually the full range of CSS3 selectors, so you can pass in as many elements as you need.  Note that once your elements are passed in, the defaults ('textarea, .RTE') will not be applied.

[For a full list of CSS selectors, see here.  The basic ones are as follows:
  1. Pound - ID   '#myDiv' refers to <div id="myDiv">.
  2. Dot - CLASS '.myDiv'  refers to <div class="myDiv">
  3. Plain text - the type of element.  'div' refers to all <div>'s
]

<!doctype>
<HTML>
    <head>
        <title>MooRTE</title>
        <script src="mootools.js"></script>
        <script src="MooRTE.js"></script>
    </head>
    <body>
        <div id="edit"> some text to edit </div>
        <textarea> some text to edit </textarea>
       
        <script> new MooRTE('#edit, textarea') </script>
    </body>
</HTML>

In the example above a rich text area ("RTE", a toolbar with editing functions) will first be applied to all divs that have an id of 'edit', then to all textareas on the page. 



That's it for basic use, there are also a few options that can be applied - 'floating', 'location', 'defaults'.
These are applied use a Javascript object as follows (all parameters are optional):

new MooRTE('#edit', {floating:true, defaults:'bold,indent', location:'elements'})


Floating refers to whether the RTE should be floating above the page (floating:true) or should be inserted into the page DOM (floating:false).  In practical use, you probably want the RTE to be a part of the structure of the page [false], but the option exists for those that want the structure to remain untouched even after the editor is applied.

Location is the intended location of the RTE.  In most cases, you want it to be set to 'elements' - ie. an editor above each element passed in.  The options are:
  1. inline - will move the toolbar around to whichever of the passed elements is selected.  Elements which have not been passed will not be made obviously editable.  [Though a few of the formatting tools are page generic.]
  2. pageTop - accross the top of the page.
  3. pageBottom - guess.
  4. none - no toolbar at all.  The elements will be made editable and shortcuts will still work (ie. ctrl+B will make the text bold).
  5. elements - applied to all passed elements.
  6. extended - will extend passed elements (or all elements, if none passed) with a .MooRTE() function.
Defaults are the default buttons.  You can pass in any list of buttons as a JSON string [or object] wherein each value is inserted into its key (see example).  All curly braces are optional.  [The available buttons are listed here, or look through MooInline.Elements in the code.]
eg:
    defaults: 'bold,strikethrough,indent'                 -    The RTE will have these three buttons     
    defaults:'Toolbar:[bold,strikethrough,indent]'     -    The RTE will have a toolbar with these buttons.

To have more control over the button display, or to create your own buttons, you must use the API, documented here.
